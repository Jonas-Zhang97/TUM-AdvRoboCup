# Documentation: Pkgs Created by Yueyang

This documentation will cover the following packages

- object_detections: object detection with YOLO V8;
- depth_segmentation: generate target object point cloud;
- env_detection: patrol mode
- pick_place: pick and place motion

## To run the packages

First, run YOLOv8 object detection:

```bash
roslaunch object_detection detection.launch
```

then, run the depth image segmentation:

```bash
rosrun depth_segmentation depth_seg.py
rosrun depth_segmentation centroid_calculator.py
```

after which, run patrol mode:

```bash
rosrun env_detection env_detection.py
```

finally, run pick and place:

```bash
roslaunch pick_place pick_place.launch
```

## object_detection

This package is created to integrate [YOLO v8 ros wrapper](https://github.com/Alpaca-zip/ultralytics_ros/tree/6db78e7dba2f8ce63b757e9f8bad271bb000ec68) into this project. It contains only a launch file that passes some customized argument to the original launch file.

## depth_segmentation

The functionality is to map the segmented mask of a specified object (generated by YOLO) onto the depth image, and after which is filtered, the depth image containing only the very object will be converted into point cloud for rge robot to grasp.

Subscribers

- */hsrb/head_rgbd_sensor/depth_registered/image*: image captured by camera on hsrb
- */hsrb/head_rgbd_sensor/depth_registered/camera_info*: intrinsic parameters of hsrb's camera
- */yolo_result*: detection result of YOLO v8
- */grasp_target_name*: name of target object (published by operater)

Publishers

- */target_depth_map*: depth image containing only the target object
- */objects_cloud*: point cloud containing only the target object

## env_detection

The function will be triggered by a flag, which is supposed to be published once when the robot arrived at a location in patrol mode. And it reads the detection result from YOLO, then the object classes will be compared with a list containing the object classes that are expected to be spotted in the current room. If anything is not in its default and optional location, this node will return the default location of that wrongly located object.

Subscribers

- */yolo_result*: detection result if YOLO
- */room_identifier*: room name published by navigation stack
- */env_detection_command*: a flag published by state machine

Publishers:

- */env_detection_error_str*: a sentence describing the wrongly located object
- */env_detection_error*: publish `False` once if anything is located wrongly
